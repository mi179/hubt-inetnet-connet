## doc-guard — 文档卫士自动同步工作流

当用户说「验证说明书」，或在修改程序逻辑代码后，必须严格执行以下流程：

### 步骤 1：侦察差异

```bash
git diff HEAD --name-only 2>/dev/null
git diff HEAD -- README.md 2>/dev/null | head -50
```

同时记录当前对话中刚刚修改过的文件列表和变更要点。

### 步骤 2：审视文档

完整读取 `README.md`，重点关注以下章节：
- ✨ 核心特性（Features 表格）
- 🖥️ 菜单展示（主菜单命令列表）
- 📋 菜单选项详解（表格）
- 命令参考（cyber-lobster 子命令表格）
- 工作流程图
- 更新日志（Changelog）

### 步骤 3：智能判定

逐条比对代码实际功能与 README 描述：

| 比对项 | 检查内容 |
|--------|----------|
| 菜单数量 | 实际菜单项 vs README 列出的菜单项 |
| 子命令 | cli.py 中注册的命令 vs README 命令表 |
| 特性 | config.py / exe_main.py 的功能 vs 特性表格 |
| 版本号 | `__init__.py` 版本 vs README 中各处版本号 |
| 新增功能 | 本次变更涉及的功能 vs README 是否覆盖 |

判定结果：
- **完全一致** → 【文档完美同步 ✅】
- **有差异** → 【文档已过期 ❌】

### 步骤 4：自动修复

如果【文档已过期 ❌】：
1. 用 `edit_file` 修正 README.md 中所有过期的描述
2. 补全缺失的功能说明、菜单选项、版本号
3. 在终端打印：
   ```
   🚨 侦测到暗箱功能变更！已自动为您同步 README.md！
   ```

如果【文档完美同步 ✅】：
1. 什么也不修改
2. 在终端打印：
   ```
   ✅ 赛博龙虾状态极佳，代码与说明书完美同步！
   ```
